# 生产回放案例：PA 出单群 2026-05-04 10:10:31

来源：`生产Case/录屏2026-05-04 10.10.31.mov`

## 场景抽象

同一微信/企微群中同时存在多客户、多车辆、多阶段作业。消息不能只按上一条顺延，必须以车辆标的、客户/合作伙伴、流程阶段和引用/焦点上下文共同判断任务归属。

## 阶段状态机

1. 新任务提交：车牌、VIN（车架号）、行驶证、身份证、营业执照等材料进入。
2. 材料收集：机器人提示补充身份证/驾驶证/营业执照，可文本提供。
3. 报价中：如“报价小能手正在努力计算中”“报价正在进行中”。
4. 报价完成待确认：输出商业险、交强险保费，并提示“出单请说确认”。
5. 确认出单：用户回复“确认”，进入投保/出单。
6. 投保中：如“投保ing，请稍后”。
7. 核保/审核中：如“交强险待审核，商业险待审核”。
8. 支付待完成：生成支付二维码。
9. 支付完成：如“支付已经完成了”。
10. 保单发送：商业险、交强险、意外险 PDF（便携式文档格式）。
11. 发票处理：发票开具中、发票 PDF（便携式文档格式）发送。

## 任务归属信号

- 强车辆信号：VIN（车架号）、车牌号、发动机号。
- 业务实体信号：车主姓名、企业名称、合作伙伴 @对象。
- 流程信号：报价流水号、Q 开头流程号、支付二维码金额/有效期、保单/发票文件名。
- 会话信号：焦点任务、最近批量任务集合、引用消息显示关联。

## 回放用例

| 编号 | 输入 | 预期 |
| --- | --- | --- |
| P001 | `确认`，上下文为报价完成待确认 | 归属焦点任务，阶段推进到投保中 |
| P002 | `非车改588`，上下文为报价任务 | 归属焦点任务，修改随车非车方案 |
| P003 | 两张未知微信图片，多个活跃任务 | 不按闲聊忽略，等待确认材料所属车辆 |
| P004 | 焦点任务已有 VIN A，当前消息出现新 VIN B | 新建独立任务，不覆盖旧任务 VIN |
| P005 | `支付已经完成了` + VIN | 归属支付阶段任务，进入保单发送 |
| P006 | `南京华夏钢铁有限公司 发票` | 归属已完成/已出保单任务，进入发票处理 |

## 新增能力要求

- 未解析附件不是闲聊：附件材料没有抽取出实体时，应进入“待解析/待确认”，不能直接忽略。
- 新 VIN 优先代表新车辆标的：除非同车牌不同 VIN 等冲突需要确认，否则不归属旧焦点任务。
- 等待确认不写事实：候选任务可展示，但不能更新旧任务槽位。
- 生产状态消息可理解：报价完成、投保中、待审核、支付完成、保单/发票发送都应作为阶段推进信号。
